```
              │    - 第三轮：语义相似      │

              ┌───────────────────────────┐
              │ 4. 计算相似度分数          │

              └─────────────┬─────────────┘
                            │

              ┌───────────────────────────┐
              │ 5. 返回 Top-K 案例          │
              │    LIMIT 5                │
              └───────────────────────────┘
```

### SQL 查询示例
```sql
-- 使用 pg_trgm 扩展进行模糊匹配
SELECT 
  project_name,
  service_type,
  equipment_type,
  fault_description,
  work_hours,
  risk_factors,
  similarity(
    fault_description, 
    $1::text
  ) AS score
FROM assessment_history
WHERE service_type = $2
ORDER BY score DESC
LIMIT 5;
```

```json

    }

```

### 错误处理
| 错误码 | 触发条件 | 处理方式 |
|--------|---------|---------|
| `DB_CONNECTION_FAILED` | PostgreSQL 连接失败 | 返回错误，检查 .env 配置 |

## 📍 阶段五：工务审核对话 (S4)

### 执行入口
```bash
cd .opencode/skills/s4-dialog-intent-detector
uv run python dialog_intent_detector.py "好的，确认"
```

### 意图类型
| 意图 | 触发词示例 | 状态转换 | 后续动作 |
|------|-----------|---------|---------|
| `CONFIRM` | "好了"/"确认"/"通过" | 待确认 → 交付 | 触发 S3 学习 |
| `EDIT` | "修改"/"调整"/"改成" | 审核中 → 审核中 | 返回 S6 修改 |
| `CANCEL` | "取消"/"放弃"/"算了" | 待确认 → 审核中 | 返回审核状态 |
| `OTHER` | 其他表述 | 保持当前状态 | 继续对话 |

### 错误处理
| 错误码 | 触发条件 | 处理方式 |
|--------|---------|---------|
| `REDIS_CONNECTION_FAILED` | Redis 连接失败 | 返回错误，检查 Redis 配置 |
| `INTENT_UNRECOGNIZED` | 无法识别意图 | 返回 OTHER，继续对话 |
| `STATE_INVALID` | 状态机转换无效 | 返回错误，重置状态 |

## 🗄️ 数据库表结构

### assessment_history (历史评估表)
```sql
CREATE TABLE assessment_history (
  id SERIAL PRIMARY KEY,
  project_name VARCHAR(255) NOT NULL,
  ship_name VARCHAR(100),
  ship_id VARCHAR(50),
  service_type VARCHAR(100),
  equipment_type VARCHAR(100),
  fault_description TEXT,
  work_hours NUMERIC(5,2),
  risk_factors JSONB,
  staff_allocation JSONB,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建 pg_trgm 索引
CREATE INDEX idx_fault_description_trgm 
ON assessment_history 
USING gin (fault_description gin_trgm_ops);
```

### learning_samples (学习样本表)
```sql
CREATE TABLE learning_samples (
  id SERIAL PRIMARY KEY,
  sample_id VARCHAR(50) UNIQUE NOT NULL,
  project_name VARCHAR(255),
  input_features JSONB NOT NULL,
  revision_action JSONB NOT NULL,
  output_preference JSONB,
  confidence NUMERIC(3,2),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### rule_candidates (规则候选表)
```sql
CREATE TABLE rule_candidates (
  id SERIAL PRIMARY KEY,
  rule_id VARCHAR(50) UNIQUE NOT NULL,
  condition TEXT NOT NULL,
  action TEXT NOT NULL,
  confidence NUMERIC(3,2),
  supporting_samples INTEGER,
  status VARCHAR(20) DEFAULT 'pending_review',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 📞 故障排查

### 常见问题

**Q1: S1 检索返回空结果**
- 检查 PostgreSQL 连接
- 确认 pg_trgm 扩展已启用
- 检查 assessment_history 表是否有数据

**Q2: S2 评估推理失败**
- 检查 references/risk_rules.json 是否存在
- 确认 PINGGU_USE_DB 配置正确
- 检查输入 JSON 格式

**Q3: S4 意图检测不工作**
- 检查 Redis 是否运行
- 确认 redis-cli ping 返回 PONG
- 检查状态机配置

**Q4: S3 学习飞轮调用失败**
- 检查初版/终版文件路径
- 确认数据库连接
- 查看日志输出
